# Prototype redesign mockups

Type: prototype
Parent: ../map.md
Blocked by: 10

## Question

Produce 2–3 cheap, visually distinct static mockups of the redesigned dashboard for the user to
react to — reaction beats specification for aesthetics.

Brief (to be sharpened by the direction grill, issue 10): each mockup is a self-contained HTML
page using real (or lightly faked) current data, distinct enough from the others that preferences
are informative. Use the `/prototype` skill; consult the `dataviz` skill before styling any
charts. These are throwaway artifacts — favor speed and visual fidelity over code quality, and do
not wire them into the production render path.

Resolve when the mockups are linked from this ticket (e.g. under
`prototypes/site-redesign-mockups/`), the user has viewed them, and their raw reactions
(what works, what doesn't, per mockup) are recorded in the answer. Picking the winner and
specifying implementation belongs to the follow-up grill (issue 12), not this ticket.
