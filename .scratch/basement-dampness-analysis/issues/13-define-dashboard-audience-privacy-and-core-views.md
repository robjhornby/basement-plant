# Define dashboard audience, privacy, and core views

Type: grilling
Status: open
Parent: ../map.md
Blocked by: 01, 05, 08, 10, 12

## Question

Who is the dashboard/report for, and which views should be included in the first publishable static version?

Use the privacy boundary from [Clarify email ingestion and hosting constraints](11-clarify-email-ingestion-and-hosting-constraints.md): raw measurement plots and derived/caveated results may be public, while raw emails, CSV files as files, exact device IDs, exact address/location, ingest address, credentials, and operational processing state stay private.

Confirm the intended audience and which views matter most: live-ish status from the latest batch, daily trends, raw measurement plots, rain overlays, leak hypothesis evidence, uncertainty-aware report values, physics explainer, raw data explorer, article embeds, or a deliberately small first page.
